# SEO Meta Tags — Required Checklist

_Last verified: 2026-03-14_

## Summary

Every page generated by Pagify must include these meta tags. This is non-negotiable — it's what separates Pagify sites from throwaway AI demos. Agents generating pages should include all of these automatically.

## Required Tags (Every Page)

```html
<head>
  <!-- Basic -->
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Page Title — Site Name</title>
  <meta name="description" content="Unique 150-160 char description of this specific page">

  <!-- Canonical URL (prevents duplicate content issues) -->
  <link rel="canonical" href="https://mysite.com/about/">

  <!-- Open Graph (Facebook, LinkedIn, iMessage previews) -->
  <meta property="og:title" content="Page Title">
  <meta property="og:description" content="Same or similar to meta description">
  <meta property="og:type" content="website">
  <meta property="og:url" content="https://mysite.com/about/">
  <meta property="og:image" content="https://mysite.com/images/og-default.jpg">
  <meta property="og:site_name" content="Site Name">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Page Title">
  <meta name="twitter:description" content="Same or similar to meta description">
  <meta name="twitter:image" content="https://mysite.com/images/og-default.jpg">
</head>
```

## Required Tags (Homepage Only)

```html
<!-- JSON-LD structured data for the organization/person -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "ProfessionalService",
  "name": "Site Name",
  "url": "https://mysite.com",
  "description": "What this practice/business does",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "City",
    "addressRegion": "State"
  }
}
</script>
```

## Required Tags (Blog Posts)

```html
<!-- Article structured data -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Post Title",
  "datePublished": "2026-03-14",
  "dateModified": "2026-03-14",
  "author": {
    "@type": "Person",
    "name": "Author Name"
  },
  "description": "Post description"
}
</script>
```

## Implementation in Eleventy

In `src/_includes/base.njk`, use template variables:

```html
<title>{{ title }} — {{ site.title }}</title>
<meta name="description" content="{{ description or site.description }}">
<link rel="canonical" href="{{ site.url }}{{ page.url }}">
<meta property="og:title" content="{{ title }}">
<meta property="og:description" content="{{ description or site.description }}">
<meta property="og:url" content="{{ site.url }}{{ page.url }}">
<meta property="og:image" content="{{ image or site.url + '/images/og-default.jpg' }}">
```

Each page's front matter provides `title` and `description`. The template handles the rest.

## Common Mistakes

1. **Same description on every page** — each page needs a unique description. "Welcome to my site" on 10 pages hurts rankings.
2. **Missing og:image** — social shares without an image get dramatically less engagement. Always have a fallback default image.
3. **Title too long** — Google truncates after ~60 characters. Keep it tight.
4. **No canonical URL** — without it, `mysite.com/about` and `mysite.com/about/` are treated as duplicate pages.
5. **Forgetting structured data** — JSON-LD is how Google generates rich snippets. It's the difference between a plain listing and one with ratings, dates, and descriptions.

## Validation Tools

- [Google Rich Results Test](https://search.google.com/test/rich-results) — validates structured data
- [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/) — validates Open Graph tags
- [Twitter Card Validator](https://cards-dev.twitter.com/validator) — validates Twitter cards

## For Coaches

When reviewing a site built by an agent:
1. View page source (Ctrl+U / Cmd+U)
2. Check that `<title>` is unique and descriptive
3. Check that `<meta name="description">` exists and is unique
4. Check that `og:image` points to a real image
5. Run the page through Google's Rich Results Test

If any of these are missing, the agent didn't finish the job.
